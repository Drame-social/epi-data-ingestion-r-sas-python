# Assumptions and Synthetic Data Notes

## Synthetic Data

All three datasets are entirely synthetic, generated by `code/python/00_generate_synthetic_data.py` using NumPy and Python's `random` module with `seed=123`. No real patient records, hospital data, county populations, or disease surveillance reports are used.

## Geographic Key Design

The 50-county FIPS universe (01001–01050, modelled on Alabama county codes) is **shared across all three datasets**. Hospital discharge records, the disease registry, and population denominators all draw county FIPS from this same set. This ensures a fully matched denominator join — a deliberate design decision mirroring real-world pipeline practice where all data sources reference the same geographic taxonomy.

## Intentional Data Quality Issues

| Dataset | Issue | Method of introduction |
|---------|-------|----------------------|
| Hospital discharge | 75 true MRN + admit_date duplicates | Overwrite last 75 rows with MRN and admit_date copied from randomly selected source rows — guarantees the deduplication check fires |
| Hospital discharge | Age sentinels (999, -1) | Direct assignment to random indices |
| Hospital discharge | Mixed sex codes (M/Male/1/2/MALE/F/Female/U) | Random recoding of 20 rows; all remaining use M/F/U |
| Hospital discharge | Invalid ZIP codes (00000) | Direct assignment to 25 random indices |
| Disease registry | Mixed date formats | Every 7th row uses MM/DD/YYYY; all others use ISO 8601 (YYYY-MM-DD) |

**Note on duplicate design:** Previous versions of this generator injected MRN duplicates without copying the admit_date, so the MRN+date deduplication check could never fire (1,500 randomly assigned dates have near-zero probability of collision). The fix copies both MRN and admit_date from the source row, ensuring true duplicate pairs.

## Population Denominators

County populations are drawn uniformly from U(5,000; 250,000) at base year 2020, with 0–2% annual growth applied for 2021–2023. They do not represent real population counts for any geographic area.

## Disease Case Counts

Cases are randomly assigned from 8 foodborne/enteric conditions. Case-fatality and hospitalisation proportions are randomly generated and do not reflect real epidemiologic patterns for these diseases.

## Hospital Discharge ICD-10 Codes

The hospital discharge file uses ICD-10 codes for general chronic conditions (diabetes, hypertension, COPD, etc.) — not the enteric diseases in the registry. This reflects a realistic administrative data context: a general discharge file is used to demonstrate ingestion and DQ pipelines, while the disease registry is the primary surveillance source for enteric incidence rates.

## What Cannot Be Verified

- Population counts bear no relation to actual US Census Bureau estimates.
- Incidence rates from this pipeline have no epidemiologic validity.
- Case-fatality and hospitalisation rates are randomly generated.
